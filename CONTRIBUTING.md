# Contributing to Dcode

Thank you for your interest in contributing to Dcode!  
We want to make coding accessible and fun, and your help is welcome.

 How to Contribute
- **Report bugs**: Open an Issue with clear steps to reproduce.
- **Suggest features**: Use the Issues tab to propose new ideas.
- **Improve docs**: Fix typos, add tutorials, or write guides.
- **Submit code**: Fork the repo, create a branch, and open a Pull Request.

 Workflow
1. Fork the repository
2. Create a new branch: `git checkout -b feature-name`
3. Make your changes
4. Commit: `git commit -m "Add feature-name"`
5. Push: `git push origin feature-name`
6. Open a Pull Request

 Guidelines
- Keep code simple and consistent with Dcode’s colon-based syntax philosophy.
- Write clear commit messages.
- Add examples or tests when possible.
- Be respectful and constructive in discussions.

Dcode is meant for schools, hobbyists, and developers who love simplicity.  
We encourage contributions from all skill levels — whether it’s fixing a typo or building a new plugin.

By contributing, you agree that your work will be licensed under the same license as the project (MIT).
